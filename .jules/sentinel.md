## 2025-02-14 - Prevent Path Traversal in Backup Generation
**Vulnerability:** The `label` parameter in `backup_dir_tree` (which can be derived from account labels/profile names) was being directly concatenated into a ZIP filename (`BACKUPS_DIR / f"{label}-{stamp}.zip"`), creating a path traversal risk if the label contained directory traversal sequences (e.g., `../../../`). Similarly, `cli_session_id` was being directly used in `rglob`, creating a risk of glob injection or traversal if `local_*.json` metadata files were manipulated maliciously.
**Learning:** Even internal metadata variables or user-configurable labels that end up in file path construction need strict sanitization, especially when generating backups or searching the filesystem via `.glob()` or `.rglob()`.
**Prevention:** Implement strict regex validation and sanitization (e.g., `re.sub(r"[^A-Za-z0-9\-\s]", "-", label)`) on any input that will be part of a file path or glob pattern to ensure it stays confined to intended directories.
