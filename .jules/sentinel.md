## 2025-02-24 - Prevent TOCTOU file permission vulnerabilities
**Vulnerability:** Files containing sensitive user data (Claude conversations, logs, backups) were being created with default permissions and subsequently restricted using `chmod` in POSIX systems. This allowed a brief Time-Of-Check to Time-Of-Use (TOCTOU) window where an attacker could access the file before its permissions were tightened.
**Learning:** In Python, standard tools like `open()`, `Path.write_text()`, and `zipfile.ZipFile(filename)` don't provide built-in mechanisms to atomically specify secure permissions (`0o600` or `-rw-------`) on file creation. Calling `.chmod(0o600)` *after* creation exposes a race condition.
**Prevention:** To prevent TOCTOU vulnerabilities during file creation, use `os.open` with the `O_CREAT | O_WRONLY | O_EXCL` flags and the explicit `0o600` mode. Then use `os.fdopen` to wrap the resulting file descriptor into a file object that can be passed to functions like `zipfile.ZipFile`. For directories, use `Path.mkdir(mode=0o700)`.
