## 2026-02-20 - [Defense Against Binary Planting/PATH Hijacking]
**Vulnerability:** The application was executing external commands like `tasklist` and `powershell.exe` using `subprocess` with relative names instead of absolute paths. This allowed execution to search through the user's `PATH` variable.
**Learning:** In a Windows environment, invoking system executables without absolute paths leaves the application vulnerable to binary planting. An attacker with minimal privileges could place a malicious executable named `tasklist.exe` or `powershell.exe` in the application's working directory or elsewhere earlier in the system's `PATH`, resulting in its execution with the privileges of the application.
**Prevention:** Always use absolute paths for executing external system commands, especially sensitive ones like `powershell.exe` or `tasklist.exe`. System commands can typically be found using `os.path.join(os.environ.get("SystemRoot", r"C:\Windows"), "System32", ...)`.
