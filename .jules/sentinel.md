## 2025-02-28 - [Path Traversal in pathlib.rglob Pattern Injection]
**Vulnerability:** Path Traversal via `cliSessionId` in `find_transcript_path`. Passing strings containing path separators (e.g. `../`) inside `pathlib.Path.rglob(f"{id}.jsonl")` allowed reading `.jsonl` files completely outside the base target directory if the provided identifier contained path traversal sequences.
**Learning:** `pathlib.Path.rglob()` does not strictly restrict the pattern to be purely a glob matching files *within* its directory hierarchy. If the pattern string contains directory separators (like `../secret`), it effectively functions as a path traversal payload starting from the rglob base path, traversing out, and then applying any remaining matching. Even though `cli_session_id`s were read from local JSON files, they were ultimately user-controlled via those files (or external attacks supplying fake indices).
**Prevention:** Always validate or sanitize input strings injected into glob patterns. For unique identifiers like UUIDs, explicitly rejecting directory separator characters (`/`, `\`, and `..`) ensures the pattern search stays bounded to the intended directory hierarchy.
